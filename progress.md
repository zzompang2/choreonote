### 결정사항 (유효)
- **대형 마켓 방향**: 유저 업로드 마켓(B안), 하이브리드 진입(C안), 3단계 로드맵
- **인증 정책**: 공유 링크는 익명 유지, 마켓 가져오기/업로드만 로그인 필요
- **업로드 단위**: 노트 전체가 아닌 특정 대형 선택 업로드 (범위 선택, 최대 5개)
- **용어 규칙**: 대형(Formation), 동선 구간(대형 사이 전환 영역), 동선(댄서 이동 경로)
- **인스펙터 버튼 규칙**: 1명 이상 선택 시 표시, 컨텍스트에 맞지 않으면 숨기지 않고 disabled
- **클라우드 저장 방향**:
  - 음악 파일은 서버에 올리지 않음 (용량 + 저작권) → 유저에게 명확히 고지
  - IndexedDB가 항상 primary, 서버는 백업/동기화 용도
  - 저장 시 단방향 업로드 (로컬→서버), 새 기기 복원은 수동
  - 충돌 감지: optimistic locking (서버 updated_at vs 로컬 cloudUpdatedAt)
  - 충돌 해결: 유저 선택 모달 (덮어쓰기 / 서버로 교체 / 둘 다 유지 / 취소)
  - 대시보드에서 로컬/클라우드 상태 뱃지로 구분
  - 비로그인 유저는 기존과 동일 (로컬 전용)
